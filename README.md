# Honeywell バーコードリーダー デバイス情報アプリ

接続中のHoneywellバーコードリーダーへメニューコマンドを直接送信し、応答を表示する専用ローカルWebアプリです。HID POS接続とUSB-COM接続に対応し、外部ツールは使用しません。

`HoneywellHidTool.exe` を起動すると、PC内のローカルHTTPサーバーを起動し、既定ブラウザで操作画面を開きます。画面はWebですが、HID/USB-COM通信はWindows側のEXEが担当します。

画面HTMLは `index.html` として分離しています。ビルド時にEXEと同じフォルダへコピーされ、起動時にそのHTMLを読み込みます。

ローカルURL：

```text
http://127.0.0.1:8765/
```

起動中は小さい管理ウィンドウが表示されます。この管理ウィンドウを閉じるとローカルHTTPサーバーも終了します。ブラウザ操作中は閉じないでください。

## 動作

1. アプリ起動時にブラウザ画面を表示し、接続中のHoneywellデバイスを検出
2. Honeywell / Hand Held Products / Barcode Scanner系デバイスを優先選択
3. HID POSまたはUSB-COM（115200 bps、8N1）で自動接続
4. 「デバイス設定出力」をクリック
5. 接続方式に合わせて設定照会コマンドを送信
6. 受信した応答を「応答結果」へ表示

応答が複数のパケットに分割された場合も、最終受信から700msまで連結して表示します。8秒以内に応答がない場合はタイムアウトを表示します。

手動コマンド欄へ入力して「コマンド送信」をクリックすると、同じ接続方式で送信します。末尾にピリオドがない場合は自動的に付加します。

USB-COM接続で `DEFALT.` を送信した場合は、USBインターフェースの切替を待って自動再検出します。切替直後のHID POSで最初のコマンドに応答がない場合は、HID接続を開き直してコマンドを1回自動再送します。

USB-COM接続時は、接続直後にバックグラウンドで `P_INFO.` を送信し、応答の `hw-sn`をシリアルNo欄へ表示します。この内部照会の応答は、画面の応答結果欄には表示しません。

HID POS接続時は、同一スキャナのHoneywell REMインターフェースが存在する場合、メニューコマンドの送受信を内部でREMへルーティングします。画面上の接続デバイス表示はHID POSのままです。

送信バイト列：

```text
SYN M CR + TERMID?;PREBK2?;SUFBK2?;DFMBK3?;PLGFOE?;PLGDCE?;REVINF.
```

画面にはHIDデバイスから取得したシリアルNoを大きく表示します。REVINF応答内にSerial情報が含まれる場合は、その値で更新します。

## ビルド

```powershell
.\build.ps1
```

生成物：

```text
bin\Release\HoneywellHidTool.exe
```

## 注意

USB-COMはHoneywellの仮想COMポートを自動検出します。HID POSはOutput Report ID `FD` を使用します。製品の機種やUSBインターフェース設定によって通信方式が異なる場合は、対象機種の仕様に合わせた調整が必要です。
